# 11｜儿童信仰问答模块使用与迭代说明

## 1. 当前版本解决什么

本模块用于家庭内学习获授权的 **《儿童信仰问答》**（*First Catechism: Biblical Truth for God’s Children*）。第一批计划导入 145 问，之后可以继续导入其他问答册。

- 中文问题与英文问题同时显示，并分别朗读。
- 孩子先口头回答，家长再揭晓中英文答案。
- 家长按“与原答案基本相同，约 80%–100%”判断 `背出来了 / 还要再背`。
- 每一次判断独立保留；同一问题同一天练两次，会产生两条记录。
- 系统按记忆阶段安排后续复习，但不把阶段、升降级等技术词显示给孩子。
- 获授权内容不会由 AI 自动翻译、缩写或改写；修正错字或标点不会删除既有练习记录。

## 2. 首次启用步骤

1. 在 Supabase SQL Editor 整段运行 [`supabase/010_catechism_learning_mvp.sql`](./supabase/010_catechism_learning_mvp.sql)。
2. 重新部署或重启本地应用。
3. 打开“家长 → 儿童信仰问答 → 导入和管理问答册”。
4. 点击“下载 CSV 模板”；下载接口会返回带 UTF-8 BOM 的文件，Excel 和浏览器均可正常识别中文。
5. 填写 145 问，选择孩子，保持“导入后立即发布”后上传。
6. 从顶部“学习模块”进入“儿童信仰问答”，点击“开始问一问”。

## 3. CSV 格式

```csv
item_key,sequence,section,question_zh,question_en,answer_zh,answer_en,scripture_reference,parent_note
child_q001,1,第一部分,"中文问题","English question","中文答案","English answer","经文出处","家长备注"
```

### 必填列

- `item_key`：问答册内稳定编号，只能使用字母、数字、下划线和短横线，例如 `child_q001`。
- `sequence`：正整数；同一问答册内不可重复。
- `question_zh`：中文问题。
- `question_en`：英文问题。
- `answer_zh`：中文答案。
- `answer_en`：英文答案。

### 可选列

- `section`：章节或主题。
- `scripture_reference`：经文出处，可包含多处经文。
- `parent_note`：学习页不展示、只在家长会话下查看的说明；由于当前孩子端也使用家长登录会话，请勿填写密码或其他敏感信息。

CSV 使用 UTF-8 编码。字段中如果有逗号、换行或双引号，应使用标准 CSV 双引号包裹；字段里的双引号写成两个双引号。单个文件最大 3MB、最多 500 问；问题、答案、章节、经文和备注还有数据库长度校验，超限时会指出 CSV 行号。每次导入都会创建独立问答册，不会按相似问题自动合并；未识别的额外列会被忽略。

## 4. 每日安排与复习算法

孩子档案拥有两项独立设置：

- `catechism_daily_new_limit`：默认每天 3 个新问题，可由家长调整。
- `catechism_review_limit`：默认每天最多 10 个到期复习，可由家长调整。

当前界面允许新问题设置为 1–20，复习上限设置为 1–50。每日边界按孩子档案的 `timezone` 计算；当天已完成的数量不会因中途改设置而倒退，保存后的设置从下一次生成页面队列开始生效，次日一定按新上限执行。

推荐顺序是：上次“还要再背”且已经到期的问题 → 逾期问题 → 阶段较低的问题 → 今日新问题。

| 当前阶段 | 背出来了 | 还要再背 |
| --- | --- | --- |
| 0（初学） | 阶段 1，1 天后 | 保持 0，次日再问 |
| 1–6 | 升一级，按新阶段间隔 | 降两级，最低 0，次日再问 |
| 7 | 保持 7，180 天后维护 | 降到 5，次日再问 |

正向间隔为：`1、3、7、14、30、60、90 天`；阶段 7 再答对后进入 `180 天`维护。

同一天可以反复练并逐次记录。为了避免连续点按虚增阶段，同日再次“背出来了”只增加次数、不再次升级；当天已经连续记过“还要再背”时，后续相同判断也不重复降级。如果当天先答对、后来答错，后来那次仍会按“答错降两级”处理。家长从问答册发起“单独练这一问”可以绕过今日数量上限，但不能绕过上述阶段保护。

“背出来了”不是考试分数。家长确认内容与原答案基本相同（约 80%–100%）即可；有停顿或个别词语不同，可以写可选备注。

## 5. 页面与管理方式

### 问一问

- 一屏一问，先问题、后答案。
- 中英文分别朗读；Azure Speech 不可用时自动使用设备系统朗读。
- 必须揭晓答案后才能提交结果。
- 答案页说明 80%–100% 的人工判断标准。

### 问答册

- 每页 20 问，支持问题、答案、英文、经文和编号搜索。
- 支持按问答册、未开始、还要再背、正在学习、稳定记住、今日到期筛选。
- 每问显示练习次数、背出次数、再背次数、阶段、最近日期和下次复习日。
- “单独练这一问”允许不等到期日也再次练习，并独立记录。

### 家长管理

- 可导入多份问答册并分配给一个或多个孩子。
- 可修正错字、标点、中英文内容、顺序、经文和家长备注。若答案含义、译本或版本发生变化，应导入一份新问答册，不要原地改写旧版本，以免历史判断失去语境。
- 问答或问答册已有历史时使用“归档”，不要从 Supabase Table Editor 直接删除。
- 取消分配只会让该孩子暂时看不到问答册；重新分配后原有学习状态仍会恢复。

## 6. 数据表与安全边界

| 表 | 作用 |
| --- | --- |
| `catechism_collections` | 问答册版本、来源、授权和发布状态 |
| `catechism_items` | 具体中英文问题与答案 |
| `learner_catechism_collections` | 问答册与孩子的分配关系 |
| `catechism_learning_states` | 当前阶段、次数和下次复习日期 |
| `catechism_attempts` | 每一次判断的不可变历史 |

所有表都开启 RLS。状态表和历史表只授予登录用户读取权，不能从网页直接插入、修改或删除；`record_catechism_attempt` 使用受限的 `SECURITY DEFINER`，在函数内再次验证 `auth.uid()`、孩子归属、问答册分配与发布状态，再原子完成状态锁定、幂等检查、历史追加和下次日期更新。函数收回了 `PUBLIC/anon` 权限，只允许 `authenticated` 调用。`request_id` 防止网络重试或双击重复升级。

Supabase Dashboard 的 SQL Editor 使用高权限角色，可能绕过 RLS；只用于迁移和只读检查，不要直接修改 `catechism_learning_states` 或 `catechism_attempts`。

本模块不需要 Supabase Edge Functions：

- 事务和复习计算：Postgres RPC。
- CSV 导入与内容维护：Next.js Server Actions。
- 中英文朗读：现有 Next.js `/api/speech` Route Handler。

## 7. 首轮验收

1. 导入模板中的 2 问，选择一个孩子并发布。
2. 确认顶部学习模块出现“儿童信仰问答”。
3. 确认中文和英文同时显示，两种朗读按钮声音语言正确。
4. 揭晓答案前不能提交结果。
5. 点“背出来了”，确认 `catechism_attempts` 新增一行，状态变为阶段 1、次日复习。
6. 单独再次练习同一问，点“背出来了”，确认同一天保留第二行但阶段不连续升级；再点“还要再背”，确认新增历史且阶段按答错规则降级一次。
7. 在问答册按“还要再背”筛选，确认次数、最近日期和下次日期正确。
8. 修改问题文字，确认历史次数没有变化。
9. 用第二个家长账号验证无法读取第一个家庭的问答册、状态和记录，也不能用第一个家庭的 learner/item ID 调用 RPC；未登录请求同样必须失败。

## 8. 后续迭代边界

暂不做儿童语音自动评分。以后若接入录音或语音转写，只能作为家长参考，不能自动覆盖 `recited/again` 事实。任何 AI 解释或延伸内容都必须存为独立候选并经家长审核，不得修改获授权的正式问题与答案。
