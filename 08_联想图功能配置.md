# 08｜汉字联想图配置（Azure `gpt-image-1-mini`）

学习卡里的“看联想图”会为当前汉字生成一张适合儿童的无文字插图。它用于建立**字义联想**，不是对汉字真实造字来源的考据。图片可随时收起，答题、复习和学习记录完全不依赖它。

## 一次性配置

1. 在 Azure AI Foundry / Azure OpenAI 中，为你的资源部署模型 **`gpt-image-1-mini`**。
2. 记下你自己设定的 **Deployment name**，它不一定等于模型名。
3. 在本机项目根目录 `.env.local` 添加或核对：

```dotenv
AZURE_IMAGE_DEPLOYMENT=你在 Azure 设置的图片部署名
AZURE_IMAGE_API_VERSION=该图片部署支持的 API 版本
```

4. 在 Vercel → Project → Settings → Environment Variables 也添加同名两项，至少勾选 **Production**；本机调试还可勾选 **Development**。
5. 重新部署 Vercel，或停止并重新执行本机 `npm run dev`，让环境变量生效。

现有的 `AZURE_OPENAI_ENDPOINT` 与 `AZURE_OPENAI_API_KEY` 会被图片接口复用，不需要新建或把密钥放进 `NEXT_PUBLIC_` 变量。

## 使用与验收

1. 以家长账号进入“学一学”。
2. 在任意汉字卡点击 **“看联想图”**。
3. 看到一张无汉字、无拼音、无水印的儿童插图后，点击 **“收起图片，再认一认”**。
4. 点击“我认识”或“再学一次”，确认学习照常进入下一张。

每次图片请求由服务器先验证：当前家长 → 当前孩子 → 当前字库 → 当前汉字。浏览器拿到的图片只临时放在当前卡片内存中；不写入 Supabase，不会改变字义、拼音或学习历史。

## 成本与使用建议

当前实现固定生成 1 张、`1024×1024`、低质量图片，按按钮时才请求。低龄识字建议把它当成“遇到难记字时的提示”，而不是每个字都点一次：一张有意义的联想图比连续刷图更有帮助，也更省成本。

OpenAI 的 GPT Image 文档将单次提示生成推荐为 Image API 场景，并列出 `gpt-image-1-mini` 的低质量方形图参考价格；实际 Azure 价格、区域可用性和 API 版本以你的 Azure 门户为准。[OpenAI Image generation guide](https://developers.openai.com/api/docs/guides/image-generation)

## 排错

| 现象 | 优先检查 |
| --- | --- |
| 页面说“联想图服务尚未配置” | 两个 `AZURE_IMAGE_*` 变量是否存在；重启本机服务或重新部署 Vercel。 |
| 页面说“服务暂时不可用” | Azure Deployment name、API version、模型区域是否正确；Azure 资源是否有图片模型权限/额度。 |
| 401/403 或“这个字不在该孩子的字库中” | 确认该孩子已导入字册，且 Supabase 已运行 `006_multi_package_library.sql`。 |
| 图片生成失败但学习可继续 | 这是预期的降级策略；先继续答题，再检查 Vercel Function Logs 中的图片 route 状态。 |

不要把 Azure Key、图片接口完整响应或包含图片的 Base64 数据粘贴进聊天、CSV、Git 或公开日志。
