# 02｜移动端 UI 与交互设计

## 1. 设计方向：像一本会复习的识字绘本

视觉不采用成人效率工具的冷白表格，也不要过度游戏化。推荐方向是 **温暖、留白、大字、低干扰**：米白底色、墨黑正文、单一草绿作为“继续/认识”的主色，橙红只用于“再练习/到期提醒”。

让人记住的核心不是花哨插画，而是每次打开都能看到一张安静、巨大的汉字卡片。

### 基础规范

| 项目 | iPhone | iPad |
| --- | --- | --- |
| 汉字主字号 | 176–224 px，随屏宽缩放 | 240–320 px |
| 正文 | 17–19 px | 18–20 px |
| 可点区域 | 不小于 48 × 48 pt | 不小于 48 × 48 pt |
| 主操作 | 固定在屏幕下部安全区上方 | 卡片底部或右侧固定区 |
| 单页焦点 | 一张卡、一个选择 | 卡片为主，右侧可显示解释/进度 |
| 横屏 | 不强制；学习卡保持单列 | 支持横屏学习工作台 |

所有儿童主流程按钮都用图标 + 文字；状态不能只依赖颜色；动画保持 150–250ms，允许系统“减少动态效果”生效。

## 2. 信息架构

### 孩子模式（iPhone 底部导航）

```text
首页             学一学             字册             我
今日任务         新字/复习队列      已学字与成就      头像与简单设置
```

- **首页**：只有“开始今天学习”这个主行动，辅以待复习数量和小小的连续学习记录。
- **学一学**：进入系统安排的队列；不让孩子在 1300 个字中迷路。
- **字册**：只看“我学过的字”，按刚学/需要再看/稳定认识筛选；不做复杂搜索。
- **我**：头像、家长入口、声音/字体偏好。

### 家长模式（iPad/桌面侧栏；iPhone 进入单独页面）

```text
概览
孩子
内容库（汉字 / 古诗 / 音乐）
导入与审核
设置
```

管理系统的“统一外壳 + 权限过滤导航”可以复用，但儿童区和家长区必须是两套导航密度。家长在 iPhone 上使用顶部返回 + 列表页面，不弹出桌面式侧栏。

## 3. 关键页面线框

### 3.1 孩子首页（iPhone）

```text
┌──────────────────────────┐
│  早上好，小满       [头像] │
│  今天我们学一点点吧          │
│                            │
│  ┌──────────────────────┐  │
│  │  今日先复习  4 个字    │  │
│  │  再认识新字  5 个      │  │
│  │                      │  │
│  │   [ 开始今天学习 ]     │  │
│  └──────────────────────┘  │
│                            │
│  已连续学习 3 天   ●●●○○    │
│  你已经稳定认识 18 个字      │
├──────────────────────────┤
│  首页       学一学   字册  我 │
└──────────────────────────┘
```

首页不要显示“正确率 57%”这种可能带来挫败感的儿童指标；家长页才显示详细比例。

### 3.2 复习卡（iPhone 的核心页面）

```text
┌──────────────────────────┐
│  ×  今日复习  3 / 9     ··· │
│                            │
│                            │
│             山              │
│                            │
│       [ 🔊 听一听 ]          │
│                            │
│     点一下看看怎么读          │
│                            │
│ ────────────────────────── │
│  [  我认识  ] [ 再学一次 ]    │
└──────────────────────────┘
```

点击“听一听”后，答案区域从下方展开，不跳到新页：

```text
 shān    声母 sh · 韵母 an · 第一声
 大山、山上
 “山上有一棵树。”
 [ 拼音小助手 ]
```

先看后答，展开后再自评。这样既能照顾刚入门的孩子，也让复习保留真正的回忆机会。

### 3.3 新字卡

与复习卡共享组件，但顶部标“今天的新朋友”。新字默认显示拼音和例词；进入下一阶段可由家长开启“先猜一猜”。同一组件可避免新字页与复习页后期产生两套逻辑。

### 3.4 完成页

只展示三件事：今天完成数、稳定认识总数、下一次见面的时间。一个“听一首奖励音频/再看字册”的次操作足够，不要立刻弹出新任务。

### 3.5 家长概览（iPad）

```text
┌──────────┬──────────────────────────────────────────────┐
│ 概览     │ 小满的学习                                     │
│ 孩子     │ [继续学习 5] [今日已完成 9] [稳定认识 48 / 1300] │
│ 内容库   │                                              │
│ 导入审核 │  本周：认识 21  ·  再学 8                      │
│ 设置     │  ▇ ▇ ▅ ▇ ▃ ▇ ▇                               │
│          │                                              │
│          │  需要多见面的字                               │
│          │  人  3 次再学  → 建议明天额外复习             │
│          │  山  2 次再学  → 已安排 1 天后                │
└──────────┴──────────────────────────────────────────────┘
```

数字与趋势为家长服务：包含“本周新字数、到期完成率、反复不稳的字、未来 7 天工作量”。不要把它做成成人 KPI 仪表板。

## 4. iPad 的两种布局

### 竖屏

延续 iPhone 卡片体验，内容最大宽度约 760px，避免卡片横向拉得过宽。家长页可显示常驻侧栏。

### 横屏学习工作台

```text
┌─────────────────────────────────────────────────────────┐
│  ← 今天学习 3 / 9                              [退出]     │
├─────────────────────────┬───────────────────────────────┤
│                         │  shān                         │
│            山            │  大山 · 山上                 │
│                         │  “山上有一棵树。”             │
│          [朗读]          │                               │
│                         │  [我认识] [再学一次]           │
└─────────────────────────┴───────────────────────────────┘
```

左侧永远是大字，右侧才出现拼音/例句/古诗提示；不要在左侧塞太多信息。横屏适合亲子共学和桌面支架使用。

## 5. 交互细节与容易忽略的体验

### 朗读

- 朗读按钮必须是明确的主控件，点击后有播放中的波纹/进度反馈，但不应遮挡汉字。
- 网络失败时显示“暂时不能朗读，先继续学习”，而不是让整张卡无法操作。
- 除汉字本音外，例词与例句播放为次级按钮，避免同时发音过多。

### 复习反馈

- 选“我认识”后：短暂的叶子/印章反馈，文案“记住啦，我们 3 天后再见”。
- 选“再学一次”后：文案“没关系，我们很快再见一次”，绝不用红叉、失败或扣分。
- 退出时自动保存当前卡的已展开状态和队列位置；未作答不计为一次尝试。

### 家长门槛

- “家长入口”可要求长按 2 秒 + PIN，防止孩子误入导入或删除页面。
- 导入、删除、重置学习记录都必须二次确认并提供影响范围。

### PWA 与 iOS

- 支持“添加到主屏幕”，配置苹果图标、启动画面和安全区。
- 学习时避免页面下拉刷新、避免浏览器滚动把底部答题按钮挤走。
- 音频只在用户点击后播放；iOS 不允许静默自动播放。
- 离线时应能看最近下载的卡片与学习记录，联网后同步；不要把“离线优先”作为 MVP 阻塞项。
- Web Push 在 iOS 上是安装为主屏幕后、且需用户授权才可用；第一版把提醒放在应用内，第二版再做可选提醒。

## 6. 从现有系统提炼的设计经验

| 已有经验 | 汉字系统的正确迁移 |
| --- | --- |
| 英语首页的继续学习卡 | 首页唯一主按钮“开始今天学习”，无需孩子选课程。 |
| 英语系统的大字号层级 | 把最大字号留给单个汉字，而不是页面标题。 |
| 英语系统固定播放器 | 改为卡片内明确的朗读按钮；不需要全局音乐播放器。 |
| 英语生词本的上下文 | 汉字“字册”也应显示例词/例句和最近学习状态。 |
| 管理系统的角色导航 | 家长/孩子权限分离，但孩子导航比管理端少得多。 |
| 管理系统的上传能力 | 只开放给家长，放入私有存储并明确文件用途。 |

## 7. 前端实现建议

- Next.js App Router + TypeScript；静态内容用服务端获取，学习队列和音频交互用小范围 Client Component。
- 把 `CharacterCard`、`AnswerReveal`、`RatingButtons`、`AudioButton`、`DailyQueue` 做成稳定的领域组件，古诗/音乐用相同的 `ReviewSession` 壳复用。
- 所有写操作先做乐观反馈，但失败后必须恢复状态并显示“尚未保存”；学习记录不能悄悄丢失。
- 采用本地中文字体或 `next/font`，避免首屏依赖外部字体；字形要以真机 iOS/iPadOS 实测为准。
- 为“儿童/家长模式”保留不同 layout，不用一套 CSS 强行兼容两种密度。

## 8. 音乐天地（已实现）

音乐模块延续现有奶油纸张、叶绿与温暖橙色，但用“乐”、音符、曲线和三种内容印记形成独立的音乐花园感。不引入黑色播放器、霓虹渐变或成人 KPI 仪表盘。

### 孩子总览

- 顶部只显示总练习次数、到期建议数和稳定掌握数。
- “今天先练”优先未练、上次没认出/没打出、已到期项，最多显示 6 张。
- 分类只有“全部 / 唱一唱 / 辨声音 / 打节奏”，不让孩子面对管理字段。

### 单项练习

- **唱一唱**：封面可空；没有封面显示轻量音符卡。原生播放器保留 iOS 无障碍和锁屏控制；下方是分行歌词、可点开的琴谱与四档结果。
- **辨声音**：先听音频，答案和乐器图不先露出；家长可填“孩子猜的是什么”，揭晓后才能点认出/没认出。
- **打节奏**：示范音频、练习说明、节奏谱都可选；结果只保留“需要再练 / 能打出”。
- 按钮文案不使用“错误、失败、降级”，但数据库会如实保留结果和新阶段。

### 家长工作台

- 先创建草稿，再进入单条编辑；发布前同时选择孩子。
- 编辑页将“文字元数据”和“MP3/图片资源”分区，家长能看见已上传文件、大小、类型和删除按钮。
- 整条内容删除会连带学习历史和 R2 文件，必须二次确认；单个资源删除只影响该 MP3/图片。
