# 字芽｜绑定 `le.fisherai6.top` 的保姆级配置教程

目标：让下面两个地址都能打开同一个字芽应用，正式对外使用第一个。

- 正式地址：`https://le.fisherai6.top`
- 当前 Vercel 地址：`https://fisher-learning-system.vercel.app`

本教程适用于：DNS 由 Cloudflare 管理、应用部署在 Vercel、账号登录使用 Supabase Auth 的当前项目。

> 这次**不需要**修改代码，也**不需要**修改 `.env.local` 中的 `NEXT_PUBLIC_SUPABASE_URL`。它必须继续是 Supabase 项目的 API 地址，绝不能改为 `le.fisherai6.top`。

## 0. 先看懂这三项分别做什么

| 平台 | 要做的事 | 最终结果 |
| --- | --- | --- |
| Vercel | 将 `le.fisherai6.top` 添加到 `fisher-learning-system` 项目 | Vercel 知道该域名应指向此应用，并签发 HTTPS 证书 |
| Cloudflare | 新增 `le` 的 CNAME 记录，指向 Vercel 给出的目标 | 访问者能找到 Vercel |
| Supabase | 允许 `https://le.fisherai6.top/auth/callback` 作为登录/确认邮箱回调 | 注册确认和登录不会被送回 localhost 或旧域名 |

## 1. 配置前检查

- [ ] 你能登录 Vercel、Cloudflare、Supabase。
- [ ] Vercel 项目名称确实是 `fisher-learning-system`，且 `fisher-learning-system.vercel.app` 当前可打开。
- [ ] Cloudflare 中 `fisherai6.top` 的 DNS 状态是 **Full**（你的截图已满足）。
- [ ] 不要删除或修改截图中已有的 `fisherai6.top`、`www`、`color`、MX、SPF、DKIM、DMARC 等记录。

你的截图中已经有一个 `color.fisherai6.top` 指向 Vercel 的例子；这次只需要额外增加一个独立的 `le.fisherai6.top`，两者不会冲突。

## 2. 先在 Vercel 添加域名（务必先做）

1. 打开 Vercel Dashboard。
2. 进入项目 **fisher-learning-system**。
3. 打开 **Settings → Domains**。
4. 在输入框中填入：

   ```text
   le.fisherai6.top
   ```

5. 点击 **Add**。
6. Vercel 会显示 DNS 配置提示。记下它给出的记录类型、名称和值。

对一个子域名，常见提示类似下面这样：

| Type | Name | Target |
| --- | --- | --- |
| CNAME | `le` | `cname.vercel-dns.com` 或 Vercel 显示的专用目标 |

> 以 **Vercel 页面实际显示的值** 为准。不要因为本教程或旧记录中出现了其他 Vercel CNAME 而自行猜测目标。

如果 Vercel 额外显示 TXT 验证记录，也一并记下，下一步在 Cloudflare 添加。

## 3. 在 Cloudflare 新增 DNS 记录

1. 打开 Cloudflare → **fisherai6.top** → **DNS** → **Records**。
2. 点击蓝色的 **Add record**。
3. 按 Vercel 第 2 步显示的要求填写。通常是：

   | 字段 | 应填写 |
   | --- | --- |
   | Type | `CNAME` |
   | Name | `le`（Cloudflare 会显示为 `le.fisherai6.top`） |
   | Target | Vercel 显示的 CNAME 目标 |
   | Proxy status | **DNS only（灰云）** |
   | TTL | `Auto` |

4. 点击 **Save**。
5. 如果 Vercel 要求 TXT 验证，再点一次 **Add record**，逐字填写 Vercel 给出的 TXT 名称和值；TXT 永远保持 DNS only。

### 为什么必须先用灰云（DNS only）

Vercel 需要先直接看到 CNAME，才能确认域名并签发证书。Cloudflare 官方也建议用于第三方验证的 CNAME 保持 DNS only。域名确认完成后，`le` 仍建议长期保持灰云：对于这个家庭学习应用已经足够稳定，且减少 Cloudflare 与 Vercel 双层 HTTPS/缓存的排查成本。

### 绝对不要做的事

- 不要为 `le` 同时创建 A、AAAA 和 CNAME；同一名称只能保留 Vercel 要求的那一条 CNAME（以及可能的 TXT 验证记录）。
- 不要把 `le` 指向 `fisher-learning-system.vercel.app`，除非 Vercel 页面明确要求该目标。
- 不要在验证阶段把 `le` 切成橙云（Proxied）。
- 不要修改根域名 `fisherai6.top` 的 A 记录，也不要修改 MX / SPF / DKIM / DMARC 邮件记录。

## 4. 回到 Vercel，等待验证和 HTTPS

1. 回到 Vercel → **Settings → Domains**。
2. 刷新或点击域名旁的验证按钮。
3. 等待状态变成 **Valid Configuration** / **Configured**。
4. 再等待 Vercel 的 SSL 状态就绪；通常是几分钟，偶尔 DNS 缓存会让它更久。
5. 浏览器打开：

   ```text
   https://le.fisherai6.top
   ```

页面正常打开且地址栏没有证书警告，才进入下一步。

如果 Vercel 显示 DNS 未配置：先确认 Cloudflare 中 Name 是 `le`，Target 与 Vercel 完全一致，且是灰云；再等待 5–15 分钟后重新检查。

## 5. 配置 Supabase Auth（登录/注册必须做）

打开 Supabase Dashboard → 对应项目 → **Authentication → URL Configuration**。

### 5.1 设置 Site URL

将 **Site URL** 改为：

```text
https://le.fisherai6.top
```

这会成为确认邮箱、重置密码等没有显式指定跳转地址时的默认正式网站地址。

### 5.2 添加 Redirect URLs

在 **Redirect URLs** 中保留现有本机/旧地址，并新增下面这条精确地址：

```text
https://le.fisherai6.top/auth/callback
```

建议最终列表至少包含：

```text
http://localhost:3000/auth/callback
https://fisher-learning-system.vercel.app/auth/callback
https://le.fisherai6.top/auth/callback
```

当前代码注册时会自动使用浏览器实际打开的域名加 `/auth/callback`。因此用 `le.fisherai6.top` 注册时，Supabase 必须允许上面这条回调地址；否则邮件确认链接可能报错或跳回旧地址。

### 5.3 不要改这些环境变量

以下变量仍维持原值：

```dotenv
NEXT_PUBLIC_SUPABASE_URL=<你的 Supabase 项目地址，通常以 supabase.co 结尾>
NEXT_PUBLIC_SUPABASE_ANON_KEY=<原值>
# 或 NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=<原值>
```

不要把 `NEXT_PUBLIC_SUPABASE_URL` 改成 `https://le.fisherai6.top`；那样应用将无法连接数据库和登录服务。

## 6. Vercel 环境变量是否需要修改？

本项目当前不需要新增正式站点 URL 的环境变量：注册回调使用浏览器当前域名自动生成，Azure 和 Supabase 的现有环境变量继续可用。

你只需要确认 Vercel 项目里原本的变量仍完整存在于 **Production** 环境：

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` 或 `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `AZURE_SPEECH_KEY`
- `AZURE_SPEECH_REGION`
- 如果启用 AI 内容：对应的 `AZURE_OPENAI_*`

域名绑定本身不需要重新填写密钥，也不需要把任何密钥复制到 Cloudflare。

## 7. 最终验收清单

建议使用 Safari 无痕窗口或另一台手机验证，避免旧登录 Cookie 干扰。

- [ ] 打开 `https://le.fisherai6.top`，页面与 Vercel 旧地址一致且没有证书警告。
- [ ] 打开 `https://le.fisherai6.top/login`，可用已有账号登录。
- [ ] 新注册一个测试邮箱；确认邮件里的链接最终回到 `https://le.fisherai6.top/auth/callback`，并能进入“学一学”。
- [ ] 从自定义域名登录后，去“册”和“家长”页确认 Supabase 数据可读写。
- [ ] iPhone Safari → 分享 → 添加到主屏幕，再从主屏幕打开，确认仍是 `le.fisherai6.top`。
- [ ] 旧地址 `https://fisher-learning-system.vercel.app` 仍能访问，作为配置期间的备用入口。

## 8. 常见故障排查

### Vercel 显示 Invalid Configuration

检查 Cloudflare 的 `le` 记录：是否是 CNAME、目标是否与 Vercel 页面完全一致、是否灰云。不要改截图中的 `color` 记录来代替 `le`。

### 浏览器显示证书错误或 Cloudflare 52x

先把 `le` 固定为 DNS only（灰云），等待 Vercel 显示域名已验证并签发证书后再测试。保持灰云即可，无需强行启用 Cloudflare 代理。

### 确认邮箱后跳转到 localhost 或旧 Vercel 地址

回到 Supabase → Authentication → URL Configuration：确认 Site URL 是 `https://le.fisherai6.top`，并且 Redirect URLs 有精确的 `https://le.fisherai6.top/auth/callback`。保存后重新发送一封新的确认邮件；旧邮件中的链接不会自动更新。

### `le.fisherai6.top` 可以打开，但登录后白屏或没有数据

这通常不是 DNS 问题。检查 Vercel Production 环境变量是否仍有 Supabase URL 和 anon/publishable key；不要把 Supabase URL 改成自定义域名。

## 9. 可选：以后是否启用 Cloudflare 橙云？

当前建议：**不启用，长期保持 DNS only**。

如果以后确实需要 Cloudflare WAF、缓存或 Bot 防护，再单独评估：先确保 Vercel HTTPS 已就绪，再将 Cloudflare SSL/TLS 模式设置为 **Full (strict)**，并逐项测试登录、Supabase 回调、iPhone PWA 和 Azure 朗读。不要使用 Flexible 模式。
